## Основные возможности

- **Каталог услуг** с описанием, ценой, длительностью и списком специалистов.
- **Пошаговая онлайн-запись**: выбор услуги → специалист → дата и время → подтверждение.
- **Автоматическая проверка занятости слотов** и защита от двойного бронирования (транзакции с уровнем изоляции Serializable).
- **Личный кабинет клиента**: просмотр, отмена и перенос записей, привязка Telegram-аккаунта.
- **Административная панель**: управление услугами, специалистами, расписанием, записями и базой знаний чат-бота.
- **Telegram-бот** с поддержкой:
  - справочной информации (`/services`, `/price`, `/address`, `/hours`);
  - поиска по базе знаний (FAQ);
  - пошаговой онлайн-записи (для пользователей с привязанным аккаунтом).
- **Разграничение доступа** по ролям: клиент, мастер, администратор.

## Архитектура и стек технологий

Трёхуровневая архитектура на основе шаблона MVC (Model‑View‑Controller).  
Приложение реализовано как монолит — веб‑сервер, сервис чат‑бота и база данных работают в едином окружении.

| Слой | Технология |
|------|------------|
| Серверная платформа | ASP.NET Core 8 (C#) |
| База данных | PostgreSQL 15 |
| ORM | Entity Framework Core 8 |
| Клиентский интерфейс | Razor Views, Bootstrap 5, JavaScript |
| Чат-бот | Telegram.Bot, Long Polling, IHostedService |
| Аутентификация | Сессионная, bcrypt |
| Тестирование | xUnit (модульное), k6 (нагрузочное), OWASP ZAP (безопасность) |
| Развёртывание | Docker Compose |

### Шаг 1. Клонирование репозитория

`bash
git clone https://github.com/your-username/SalonBooking.git
cd SalonBooking
`

### Шаг 2. Настройка конфигурации

Создайте в корне проекта файл appsettings.Development.json (для локального запуска) или настройте переменные окружения.

Пример appsettings.Development.json:

`json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Database=salonbooking;Username=salon_admin;Password=securePass123"
  },
  "TelegramBot": {
    "Token": "ВАШ_ТОКЕН_БОТА"
  }
}
`

Важно: не коммитьте реальные токены в репозиторий. Для продакшена используйте переменные окружения.

### Шаг 3. Запуск через Docker Compose (рекомендуется)

`bash
docker compose up -d
`

Приложение будет доступно по адресу [http://localhost:5000](http://localhost:5000).

### Шаг 4. Локальный запуск без Docker

Убедитесь, что PostgreSQL запущен и база данных salonbooking создана.

Примените миграции:

`bash
dotnet ef database update
`

Запустите приложение:

`bash
dotnet run
`

Приложение будет доступно по адресу [https://localhost:5001](https://localhost:5001) (или http://localhost:5000 в среде разработки).  
При первом запуске база данных автоматически заполняется тестовыми данными (услуги, специалисты, расписание, база знаний бота).

## Ограничения прототипа и план доработок

Текущая версия является прототипом и содержит осознанные упрощения. Перед промышленной эксплуатацией необходимо:

- Реализовать политику повторных попыток (retry policy) для Telegram‑бота.
- Внедрить недостающие заголовки безопасности (Content‑Security‑Policy, HTTP Strict Transport Security, X‑Content‑Type‑Options).
- Вынести хранение состояния диалогов бота из оперативной памяти в распределённый кэш или БД.
- Заменить ручной ввод Telegram Chat ID на безопасный механизм привязки через одноразовый код.
- Провести нагрузочное тестирование на выделенном стенде для подтверждения пропускной способности.
- Настроить автоматизированное резервное копирование и централизованный сбор логов.
